Methodology

What was sent

  • Transaction shape: compute budget instructions plus a memo-free self-transfer.
  • Compute unit limit: 660.
  • Priority fee: 0 micro-lamports/CU.
  • Duration: 3600 seconds, leader-paced.

How landing is measured

  • submit_to_deshred: gateway receive timestamp to first deshred stream observation.
  • submit_to_processed: gateway receive timestamp to Yellowstone processed transaction metadata.
  • landed_slot_delta: landed slot minus sender-observed slot at submission.
  • processed_block_index: transaction index inside the processed block.
Important: ACK latency is only the relay/provider response path. It does not prove leader admission or transaction landing. Product claims should use deshred or processed observations matched by signature.

Performance Dimensions

This report currently has one selected provider/route, so a normalized 0-100 provider score would be misleading. For multi-provider or route-isolated runs, RPCEdge should score the same core buckets used by serious landing benchmarks: landing latency in milliseconds, landed slot offset, block position, same-slot landing rate, and overall success ratio. Tail latency should be weighted more heavily than averages because HFT users care about the bad windows.
